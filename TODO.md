# Tomorrow TODO

## 목표

내일의 목표는 기능을 더 많이 붙이는 것이 아니라, 문과/개발 노베이스 사용자가 겁먹지 않고 시작할 수 있게 만드는 것입니다.

핵심 작업:

> 온보딩 + 용어 번역 + 결과물 미리보기

## 1. 첫 화면 온보딩 만들기

현재는 바로 Wizard가 시작됩니다. 내일은 첫 화면을 먼저 보여줘야 합니다.

첫 화면에 들어갈 내용:

- 이 서비스가 무엇을 해주는지
- 개발 지식이 없어도 된다는 안내
- 선택지만 골라도 된다는 안내
- 끝나면 무엇을 받는지
- 예상 소요 시간
- 시작하기 버튼

추천 문구:

```text
아이디어만 있어도 괜찮아요.
몇 가지 선택지를 고르면 AI 개발 도구에 바로 넣을 수 있는 기획서와 개발 프롬프트를 만들어드립니다.
```

완료 기준:

- 첫 진입 시 온보딩 화면이 보인다.
- 시작하기를 누르면 10단계 Wizard로 이동한다.
- 새로고침해도 사용자가 작성 중이면 Wizard 상태를 유지할지 결정한다.

## 2. 결과물 미리보기 추가

초보자는 끝까지 하면 무엇을 받는지 알아야 안심합니다.

온보딩 화면 또는 Wizard 상단에 결과물 미리보기를 추가합니다.

미리보기 항목:

- 서비스 요약서
- MVP 기능 명세
- 화면 구조
- 기술 설계
- AI 개발 프롬프트
- 배포 체크리스트

완료 기준:

- 사용자가 시작 전에 최종 결과물을 이해할 수 있다.
- 결과물 이름이 너무 개발자스럽지 않다.

## 3. 용어 번역 사전 만들기

전문용어를 쉬운 말로 바꾸는 데이터 구조를 만듭니다.

파일 후보:

```text
src/lib/glossary.ts
```

초기 용어:

- MVP = 처음 만들 최소 기능
- API = 다른 서비스와 데이터를 주고받는 통로
- 아키텍처 = 서비스를 어떤 구조로 만들지
- 기술 스택 = 개발에 쓸 도구 조합
- 배포 = 인터넷에 공개하기
- 프롬프트 = AI 개발자에게 줄 작업 요청서
- 환경변수 = 비밀값을 안전하게 보관하는 설정
- LocalStorage = 내 브라우저에 임시 저장하기
- Vercel = 만든 웹사이트를 인터넷에 올리는 서비스
- Supabase = 로그인과 데이터 저장을 쉽게 붙이는 서비스

완료 기준:

- 용어와 쉬운 설명이 코드 데이터로 분리되어 있다.
- UI에서 재사용할 수 있다.

## 4. 단계명 초보자용으로 바꾸기

현재 단계명을 초보자 중심으로 바꿉니다.

변경안:

- 아이디어 입력 → 만들고 싶은 것 고르기
- 타겟 · 문제 정의 → 누가 왜 필요할지 정하기
- 서비스 형태 선택 → 어떤 모양의 서비스인지 고르기
- 레퍼런스 분석 → 참고할 서비스 고르기
- MVP 기능 정리 → 처음 만들 기능 고르기
- 화면 구조 설계 → 어떤 화면이 필요할지 정하기
- 데이터 · API 설계 → 저장할 정보 정하기
- 아키텍처 추천 → 만들 방법 정하기
- 프롬프트 생성 → AI에게 시킬 말 만들기
- 배포 · 운영 체크리스트 → 인터넷에 공개하기 전 확인하기

완료 기준:

- 화면에서 전문용어보다 쉬운 표현이 먼저 보인다.
- 전문용어는 설명이나 도움말에서 보조로 다룬다.

## 5. 질문 옆 도움말 개선

각 질문에는 “왜 이걸 묻는지”가 있어야 합니다.

예:

```text
왜 묻나요?
누가 사용할지 알아야 기능을 작게 줄이고, 첫 버전에서 꼭 필요한 것만 고를 수 있어요.
```

완료 기준:

- 각 단계 도움말이 초보자 말투로 바뀐다.
- 사용자가 정답을 맞혀야 한다는 느낌을 받지 않는다.

## 6. 결과 화면 설계 시작

아직 구현은 작게 시작합니다. 먼저 결과 화면 구조만 잡습니다.

탭 후보:

- 요약서
- 처음 만들 기능
- 화면 구조
- 만들 방법
- AI에게 시킬 말
- 공개 전 체크리스트

완료 기준:

- 10단계 완료 시 결과 화면으로 이동하는 구조가 생긴다.
- 탭 UI의 기본 틀이 있다.
- 광고 슬롯이 들어갈 위치를 CSS/컴포넌트 구조로 미리 비워둔다.

## 7. 광고 슬롯 설계

실제 광고 적용은 나중에 하더라도, 위치와 컴포넌트 구조는 먼저 잡습니다.

파일 후보:

```text
src/components/monetization/AdSlot.tsx
```

환경변수 후보:

```bash
NEXT_PUBLIC_ADSENSE_CLIENT_ID=""
NEXT_PUBLIC_ADSENSE_SLOT_RESULT_TOP=""
NEXT_PUBLIC_ADSENSE_SLOT_RESULT_BOTTOM=""
```

완료 기준:

- 광고가 없어도 레이아웃이 깨지지 않는다.
- 결과 화면 상단/하단에 광고 슬롯 위치가 있다.
- 실제 광고 ID는 `.env.local`에만 넣는 구조다.
