# Vibe Coding Navigator Development Document v2

## 1. v2의 목적

v2의 핵심은 초보자가 서비스 아이디어를 잃어버리지 않고 여러 초안을 저장, 선택, 이어서 완성할 수 있게 만드는 것입니다.

v1이 “아이디어를 AI 개발 프롬프트로 바꾸는 10단계 Wizard”였다면, v2는 다음 방향으로 확장합니다.

- 첫 화면에서 서비스 목적을 쉽게 이해한다.
- 질문마다 추천 선택지를 클릭해 입력한다.
- 어려운 개발 용어는 쉬운 말로 번역한다.
- 10단계 이후에는 문서형 결과물을 탭으로 받는다.
- 작성 중 화면과 결과 화면에 수익화 가능한 광고 슬롯을 둔다.
- 여러 개의 입력 초안을 저장하고, `이어하기`에서 골라 다시 연다.

## 2. 현재 구현된 기능

현재까지 구현된 기능은 다음과 같습니다.

- Windows 95 스타일의 10단계 Wizard UI
- 초보자용 첫 화면 온보딩
- 10단계 완료 후 받을 결과물 미리보기
- 추천 선택지 클릭 입력
- 단계별 다중 입력 필드
- 단계별 도움말 문구
- 오른쪽 현재 단계 프롬프트 패널
- 오른쪽 현재까지 입력된 내용 요약 패널
- LocalStorage 자동 저장
- 전체 개발 프롬프트 생성
- 10단계 완료 후 결과 탭 화면
- 결과 탭별 복사 기능
- 결과 화면 상단/하단 광고 슬롯 placeholder
- Wizard 진행 중 하단 광고 슬롯 placeholder
- `.env.local` 기반 로컬 환경변수 관리

## 3. 상단 버튼 기능 정리

상단 버튼은 사용자가 프로젝트를 관리하는 기본 행동입니다. 현재 동작과 v2에서 확장할 동작을 분리합니다.

| 버튼 | 현재 동작 | v2 확장 방향 |
|---|---|---|
| 새 프로젝트 | 현재 입력값과 LocalStorage 저장값을 초기화하고 첫 화면으로 돌아간다. | 초기화 전 확인 모달을 띄운다. 기존 초안을 삭제할지, 새 초안으로 분리할지 선택하게 한다. |
| 이어하기 | Wizard 화면으로 이동한다. LocalStorage에 저장된 마지막 입력값을 계속 작성한다. | 저장된 여러 프로젝트 초안 목록을 보여주고, 사용자가 하나를 클릭해 열게 한다. |
| 저장 | 현재 프로젝트를 LocalStorage에 저장한다. 입력할 때 자동 저장도 함께 동작한다. | 프로젝트 이름, 마지막 수정 시간, 진행 단계, 완료율과 함께 저장한다. 저장 완료 피드백을 보여준다. |
| 내보내기 | 현재까지의 전체 개발 프롬프트를 클립보드에 복사한다. | 현재 탭 Markdown 다운로드, 전체 결과물 Markdown 다운로드, Codex/Cursor/Claude용 프롬프트 내보내기로 확장한다. |
| 도움말 | 현재는 버튼 UI만 있고 클릭 동작은 없다. | 초보자용 용어 사전, 현재 단계 설명, “왜 이 질문을 묻는지”를 보여주는 도움말 패널을 연다. |

상단 메뉴의 `파일`, `편집`, `보기`, `도구`, `도움말`은 현재 Windows 95 분위기를 위한 장식 텍스트입니다. v2 이후 실제 드롭다운 메뉴로 확장할 수 있습니다.

## 4. 이어하기 기능 v2

사용자가 말한 “이어하기를 보면 여러 입력값 중 하나를 클릭해서 볼 수 있으면 좋겠다”는 방향을 v2의 핵심 저장 기능으로 정의합니다.

### 4.1 사용자가 기대하는 흐름

1. 사용자는 여러 서비스 아이디어를 작성할 수 있다.
2. 각 아이디어는 하나의 프로젝트 초안으로 저장된다.
3. `이어하기` 버튼을 누르면 저장된 프로젝트 목록이 열린다.
4. 사용자는 목록에서 원하는 프로젝트를 클릭한다.
5. 해당 프로젝트의 입력값, 현재 단계, 결과 문서가 다시 열린다.

### 4.2 이어하기 목록에 보여줄 정보

각 저장 프로젝트 카드는 다음 정보를 보여줍니다.

- 서비스 이름
- 한 줄 설명 또는 서비스 형태
- 마지막 수정 시간
- 현재 진행 단계
- 완료율
- 주요 키워드
- 열기 버튼
- 삭제 또는 보관 버튼

초보자에게는 파일 관리처럼 느껴지면 어렵기 때문에, 표현은 “저장된 아이디어” 또는 “작성 중인 아이디어”가 적합합니다.

추천 화면 제목:

> 작성 중인 아이디어를 골라 이어서 만들어요

추천 빈 상태 문구:

> 아직 저장된 아이디어가 없어요. 새 프로젝트를 눌러 첫 아이디어를 만들어보세요.

### 4.3 저장 데이터 구조 방향

현재는 하나의 프로젝트만 LocalStorage에 저장합니다. v2에서는 여러 프로젝트를 저장할 수 있도록 구조를 바꿉니다.

예상 구조:

```ts
type SavedProject = {
  id: string;
  name: string;
  project: Project;
  currentStep: number;
  createdAt: string;
  updatedAt: string;
};
```

LocalStorage key 방향:

```bash
NEXT_PUBLIC_STORAGE_KEY="vibe-coding-navigator:v2"
```

저장 방식:

- `projects`: 여러 프로젝트 배열
- `activeProjectId`: 마지막으로 열었던 프로젝트 ID
- 기존 v1 단일 저장값이 있으면 첫 v2 프로젝트로 마이그레이션

## 5. 광고와 ROT 방향

ROT(Return on Token)를 높이려면 사용자가 쓴 시간과 토큰 대비 바로 실행 가능한 결과물을 받아야 합니다. 동시에 서비스가 계속 운영되려면 수익화 지점이 필요합니다.

현재 광고 슬롯 방향:

- Wizard 진행 중 하단 광고 슬롯
- 결과 화면 상단 광고 슬롯
- 결과 화면 하단 광고 슬롯

운영 원칙:

- 질문 입력을 방해하는 위치에는 광고를 과하게 넣지 않는다.
- 결과 확인, 복사, 다운로드처럼 체류 시간이 긴 구간에 광고를 둔다.
- 광고가 없어도 레이아웃이 깨지지 않게 placeholder를 유지한다.
- 추후 AdSense 환경변수로 실제 슬롯을 연결한다.

환경변수 방향:

```bash
NEXT_PUBLIC_ADSENSE_CLIENT_ID=""
NEXT_PUBLIC_ADSENSE_SLOT_WIZARD_BOTTOM=""
NEXT_PUBLIC_ADSENSE_SLOT_RESULT_TOP=""
NEXT_PUBLIC_ADSENSE_SLOT_RESULT_BOTTOM=""
```

## 6. v2 우선 개발 순서

v2에서 가장 먼저 개발할 순서는 다음과 같습니다.

1. 여러 프로젝트 저장 구조 만들기
2. `이어하기` 클릭 시 저장된 프로젝트 목록 열기
3. 프로젝트 카드 클릭 시 해당 입력값 불러오기
4. 새 프로젝트 생성 시 기존 초안을 보존할지 확인하기
5. 저장/내보내기/도움말 버튼에 피드백 추가하기
6. Markdown 다운로드 기능 추가하기
7. 도움말 버튼을 용어 사전과 연결하기
8. 광고 슬롯을 환경변수 기반 구조로 고도화하기

## 7. 내일 개발 목표

내일의 핵심 목표는 `이어하기` 기능을 진짜 프로젝트 선택 기능으로 만드는 것입니다.

완료 기준:

- 여러 개의 프로젝트 초안을 저장할 수 있다.
- `이어하기` 버튼을 누르면 저장된 아이디어 목록이 열린다.
- 목록에서 하나를 클릭하면 해당 프로젝트가 Wizard에 로드된다.
- 기존 단일 저장값이 있으면 자동으로 v2 저장 구조에 옮겨진다.
- 저장된 프로젝트가 없을 때도 초보자가 이해할 수 있는 빈 상태 안내가 나온다.
- `npm run lint`와 `npm run build`가 통과한다.
