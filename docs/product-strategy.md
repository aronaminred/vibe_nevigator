# Vibe Coding Navigator Product Strategy

## 1. 서비스의 핵심 방향

Vibe Coding Navigator는 바이브 코딩 초보자가 서비스 아이디어를 개발 가능한 구조로 바꾸도록 돕는 10단계 Wizard다.

핵심 가치는 단순한 추천이 아니라, 사용자가 바로 Codex, Cursor, Claude, ChatGPT 같은 AI 개발 도구에 전달할 수 있는 실행 가능한 결과물을 만드는 것이다.

정의:

> 선택형 진단 도구 + 실시간 기획 문서 생성기 + AI 개발 프롬프트 빌더

## 2. 참고 서비스에서 배운 점

참고 서비스:

- https://apptype.technote.wiki/
- https://apptype.technote.wiki/index-v2

두 서비스의 주요 특징:

- 초보자에게 빈 입력칸을 주지 않고 선택지 카드로 시작한다.
- 질문 수를 제한해 부담을 낮춘다.
- 사용자는 자신의 상황과 가장 가까운 선택지만 고르면 된다.
- 답변을 점수화하거나 분류해 추천 결과를 만든다.
- 결과 화면에서 앱 유형, 기술 스택, 추천 이유, 다음 행동을 제공한다.
- 회원가입 없이 무료로 바로 사용 가능하다.
- 결과 화면과 체류 시간을 광고 수익화 지점으로 활용한다.

우리 서비스에 적용할 원칙:

- 모든 단계는 추천 선택지를 먼저 보여준다.
- 선택지를 누르면 입력 초안이 자동 작성된다.
- 사용자는 직접 작성보다 수정과 보완에 집중한다.
- 각 단계 옆에는 현재 단계 프롬프트가 실시간 생성된다.
- 마지막에는 전체 개발 프롬프트와 문서형 결과물이 산출된다.

## 3. 10단계 이후 산출물

10번 단계 이후 사용자는 단순한 텍스트 하나가 아니라 개발 실행 패키지를 받아야 한다.

필수 산출물:

1. 서비스 기획 요약서
   - 서비스 이름
   - 한 줄 설명
   - 타겟 사용자
   - 해결하려는 문제
   - 핵심 가치
   - 제외할 기능

2. MVP 기능 명세서
   - 반드시 만들 기능
   - 나중에 만들 기능
   - 화면별 기능
   - 사용자 액션
   - 완료 기준

3. 화면 구조 설계서
   - 필요한 화면 목록
   - 각 화면의 역할
   - 주요 UI 컴포넌트
   - 사용자 흐름

4. 기술 설계서
   - 추천 기술 스택
   - 데이터 구조
   - 저장 방식
   - API 필요 여부
   - 배포 방식
   - 환경변수 체크리스트

5. AI 개발 프롬프트
   - 구현 범위
   - 파일 구조
   - 디자인 방향
   - 보안 조건
   - QA 기준
   - README, 커밋, 배포 지시

6. 배포 및 운영 체크리스트
   - 빌드 확인
   - 모바일 화면 확인
   - 환경변수 등록
   - 배포 URL 확인
   - 복사 및 다운로드 동작 확인

결과 화면은 탭 구조가 적합하다:

- 요약서
- MVP 명세
- 화면 구조
- 기술 설계
- 개발 프롬프트
- 배포 체크리스트

각 탭에는 다음 액션을 둔다:

- 복사
- Markdown 다운로드
- 새 프로젝트 시작

## 4. ROT(Return on Token) 관점

ROT는 사용자가 쓴 토큰, 시간, 집중력 대비 얼마나 실행 가능한 결과를 얻는지를 뜻한다.

이 서비스에서 ROT를 높이는 방법:

- 사용자가 직접 긴 글을 쓰지 않게 한다.
- 선택지만 눌러도 문서 초안이 만들어지게 한다.
- 단계별 프롬프트와 전체 프롬프트를 동시에 제공한다.
- 결과물이 바로 AI 개발 도구에 붙여넣을 수 있어야 한다.
- 최종 결과가 개발문서, 명세, 프롬프트, 배포 체크리스트까지 포함해야 한다.

사용자가 느껴야 하는 최종 상태:

> 내 아이디어가 개발 가능한 요구사항 문서와 AI 개발 프롬프트로 바뀌었다.

## 5. 수익화 전략

이 서비스는 무료 진단 도구로 시작하고, 트래픽과 결과 화면 체류 시간을 수익화한다.

### 5.1 광고 기반 수익화

초기 수익화 모델은 광고가 적합하다.

추천 광고 위치:

- 10단계 완료 후 결과 화면 상단
- 결과 탭 사이 하단
- 전체 개발 프롬프트 아래
- Markdown 다운로드 버튼 근처

주의할 점:

- 입력 도중에는 광고를 과하게 넣지 않는다.
- 사용자의 사고 흐름을 끊지 않는다.
- 결과 화면처럼 체류 시간이 긴 구간에 배치한다.

환경변수 예시:

```bash
NEXT_PUBLIC_ADSENSE_CLIENT_ID=""
NEXT_PUBLIC_ADSENSE_SLOT_RESULT_TOP=""
NEXT_PUBLIC_ADSENSE_SLOT_RESULT_BOTTOM=""
```

### 5.2 프리미엄 기능

광고만으로는 한계가 있으므로 이후 유료 기능을 고려한다.

가능한 프리미엄 기능:

- 광고 제거
- 프로젝트 저장
- 결과 문서 PDF 다운로드
- Markdown 다운로드
- Codex/Cursor/Claude용 고급 프롬프트
- 서비스 유형별 템플릿
- AI 자동 분석
- URL/이미지 기반 레퍼런스 분석

### 5.3 제휴 수익

이 서비스와 어울리는 제휴 후보:

- Vercel
- Supabase
- Cursor
- Claude
- OpenAI API
- 도메인 서비스
- 호스팅 서비스
- 노코드/로우코드 툴

## 6. 다음 개발 우선순위

1. 10단계 완료 후 결과 화면 분리
2. 결과 화면 탭 구조 구현
3. 각 탭별 Markdown 생성 로직 구현
4. 광고 슬롯 컴포넌트 설계
5. 광고 환경변수 추가
6. Markdown 다운로드 기능 추가
7. 최종 결과 복사 UX 개선
8. Vercel 배포 및 광고 적용 준비

## 7. 제품 원칙

- 초보자에게 빈칸부터 보여주지 않는다.
- 선택지를 먼저 주고, 수정할 수 있게 한다.
- 모든 선택은 결과 문서와 프롬프트에 반영되어야 한다.
- 결과는 추천에서 끝나지 않고 실행 가능한 개발 지시문이어야 한다.
- 수익화는 사용 흐름을 방해하지 않고 결과 화면 중심으로 배치한다.
- 무료 사용자는 충분한 가치를 얻고, 고급 사용자는 저장/다운로드/고급 프롬프트에 비용을 지불하게 한다.
